Claude Code Dynamic Workflows 深度解读

2026.5.31 · Claude Opus 4.8 · 1,000子Agent并行 · Bun 75万行移植案例

一句话总结

Anthropic 在 5 月 28 日发布了 Claude Code 有史以来最大的更新——Dynamic Workflows（动态工作流）。这不是普通的Agent升级，而是让AI能自己规划任务、拆解子任务、调度多达1,000个子Agent并行执行，最后汇总结果。Bun的创建者用它11天完成了从Zig到Rust的75万行代码移植，通过率99.8%。

什么是 Dynamic Workflows？

传统的AI对话是这样：你问 → 模型回答。再好一点：你给一个复杂任务 → 模型一步步完成。

Dynamic Workflows 完全不一样：

  • 你描述一个大型任务 → Claude 自动规划
  • 它写一个 JavaScript 脚本 来编排子Agent
  • 运行时在后台执行脚本，你的会话保持响应
  • 计划存在于脚本变量中，而不是Claude的上下文窗口里
  • 只有最终结果回到你的会话

关键是：子Agent之间可以相互审查。一批Agent从不同角度解决问题，另一批Agent试图反驳它们的结论。迭代收敛后，结果经过验证才返回给你。

震撼案例：Bun从Zig移植到Rust

Dynamic Workflows 最令人震撼的实测案例来自 Bun 的作者 Jarred Sumner：

📊 75万行 Rust 代码生成

✅ 99.8% 现有测试通过

⏱️ 11天 从首次提交到合并

🤖 数百个 Agent 并行工作，每个文件2个审阅Agent

具体流程：

  1. 第一步： 一个Workflow为每个struct字段映射正确的Rust生命周期
  2. 第二步： 另一个Workflow写出每个.rs文件作为行为一致的移植
  3. 第三步： 数百个Agent并行工作，每个文件配备2个审阅Agent
  4. 第四步： 自动修复循环驱动构建+测试，直到全部通过

这个案例的意义在于：AI已经在做人类程序员需要几个月的架构级移植工作。11天、75万行、99.8%通过率——数据不会说谎。

如何上手？三种触发方式

方式一：关键词触发

在 Prompt 中包含 "workflow" 一词即可：

Run a workflow to audit every API endpoint under src/routes/ for missing auth checks

方式二：Ultracode 模式

让Claude自行决定何时使用Workflow：

/effort ultracode

方式三：内置 Deep Research

/deep-research What changed in the Node.js permission model between v20 and v22?

成本和注意事项

⚠️ 重要：成本会显著增加

Dynamic Workflows 和 Fast Mode 都会大幅消耗 Token。

  • 一次运行最多可启动1,000个Agent
  • Fast Mode 按使用额度计费（不是套餐额度）
  • 建议先用小范围任务测试，观察用量后再扩展
  • 作为 Research Preview，定价和可用性可能变更
